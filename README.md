### Описание проекта
Анализ данных из базы вакансий сайта hh.ru

### Какой кейс решаем?
Кадровому агенству, которое подбирает вакансии для IT-специалистов, нужно создать модель машинного обучения, которая будет рекомендовать вакансии клиентам агентства, претендующим на позицию Data Scientist. Нам необходимо в базе вакансий сайта hh.ru провести анализ данных, понять что они из себя представляют и насколько соответствуют изначальной задаче. Весь анализ будет проводиться с помощью запросов СУБД - PostgreSQL.

### Краткая информация о данных
Данные представлены в виде 5 связанных таблиц:

<img src = https://lms.skillfactory.ru/asset-v1:SkillFactory+DST-3.0+28FEB2021+type@asset+block@SQL_pj2_2_1.png alt="drawing" style="width:400px;">

1. ***VACANCIES*** - данные по вакансиям
    - *id* - id вакансии
    - *name* - название ваканси
    - *key_skills* - ключевые навыки
    - *schedule* - тип рабочего графика
    - *experience* - требования к опыту
    - *employment* - тип трудоустройства
    - *salary_from* - нижняя граница зарплатной вилки
    - *salary_to* - верхняя граница зарплатной вилки
    - *area_id* - id региона вакансии
    - *employer_id* - id работодателя

2. ***AREAS*** - данные по регионам
    - *id* - id региона
    - *name* - название региона

3. ***EMPLOYERS*** - данные по работодателям
    - *id* - id работодателя
    - *name* - название работодателя
    - *area* - регион регистрации

4. ***INDUSTRIES*** - данные по сферам деятельности работодателей
    - *id* - id сферы деятельности
    - *name* - название сферы деятельности

5. ***EMPLOYERS_INDUSTRIES*** - доп.таблица для организации связи "работодатели-сферы деятельности"
    - *employer_id* - id работодателя
    - *industry_id* - id сферы деятельности

### Этапы работы над проектом
1. Предварительный анализ данных
2. Детальный анализ вакансий
3. Анализ работодателей
4. Предметный анализ

### Результаты
- Определен возможный будущий целевой признак.
- Определена проблематика данных в вакансиях.
- Проведен детальный общий анализ и предметный анализ данных.
